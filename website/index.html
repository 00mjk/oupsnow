A Bug tracker, written with Merb and using DataMapper like ORM. Oupsnow manage several project.

Features

  • Manage Several project
  • Manage ticket project by project

Requirements

Currently you need all of those things to get Oupsnow to run:

  • Merb 1.0.8
  • Ruby of 1.8.6
  • DataMapper 0.9.9
  • A database supported by DataMapper (MySQL, SQLite3, PostgreSQL ...)

Installing

With the tar.gz or any other archive:

  • Extract sources to a folder
  • Create a database.yml file in the config directory. Generate by example by rake db:database_yaml
  • Create your database with tool on your database
  • Generate the good schema in your database: MERB_ENV="production" rake db:automigrate
  • Add first data in your database: MERB_ENV="production" thor oups_now:bootstrap:first_value
  • Start the server in production mode: merb -e production

Information about this project

Oupsnow is actually consider like an Beta version, and is under development.

All contributions are welcome.

I suck in design, I know it and I am sorry but I will really be happy if anyone could help me.

If you want to contribute, all work is made under a git repository. You can clone the source with the following command :

git clone git://github.com/shingara/oupsnow.git

After a clone you need update the submodule :

git submodule init git submodule update

A redmine development platform is used. Feel free to post your feature requests and defects report. In futur version, Oupsnow to be used to manage this project
